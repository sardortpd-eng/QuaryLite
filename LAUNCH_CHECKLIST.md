# QuaryLite — OSS Launch Checklist (MIT)

Last updated: 2026-04-29

---

## ✅ Done

### A — Security

- [x] **Sanitize AI markdown** — `DOMPurify.sanitize(marked(part))` wraps every AI response render (`src/components/ChatPane.tsx`). Only runs after streaming ends to avoid per-token overhead.
- [x] **Content Security Policy** — `tauri.conf.json` CSP was `null`; now restricts scripts to `'self'` and connects only to known AI provider origins.
- [x] **AI write confirmation** — Write and DDL SQL blocks generated by AI show an Approve / Reject card before executing. Default mode is `confirm`. `aiWriteMode` prop supports `off | confirm | auto`.
- [x] **SQL deny-list** — `check_sql_safety()` in `lib.rs` blocks `ATTACH DATABASE`, `PRAGMA WRITABLE_SCHEMA`, `LOAD_EXTENSION` before any query runs.
- [x] **30s busy timeout** — `conn.busy_timeout(Duration::from_secs(30))` added to all new connections.
- [x] **Mutex poison recovery** — `lock_tx()` helper replaces all `state.0.lock().unwrap()` calls; a panicked prior lock no longer crashes the process.
- [x] **Keyring error surfacing** — `keyring_set()` now returns `Result<(), String>`; `save_settings` propagates the error to the frontend so a failed keychain save is visible.

### B — Identity & Branding

- [x] **`tauri.conf.json` rewrite** — `productName` → `"QuaryLite"`, `identifier` → `"com.querylite.app"`, `title` → `"QuaryLite"`, added `publisher`, `copyright`, `category`, `shortDescription`, `longDescription`
- [x] **`src-tauri/Cargo.toml`** — added `description`, `authors`, `license = "MIT"`, `repository`, `homepage`, `readme`, and `[profile.release]` with LTO + strip
- [x] **`package.json`** — added `description`, `author`, `license`, `repository`, `homepage`, `bugs`
- [x] **`LICENSE`** — MIT license file created at repo root (© 2026 QuaryLite)
- [x] **`README.md`** — replaced Vite template with tagline, feature list, install instructions (including Gatekeeper/SmartScreen workaround), keyboard shortcuts table, AI provider table, contributing link, MIT badge

### D — UX Credibility

- [x] **Humanize errors** — `humanizeError()` helper in `CenterPane.tsx`. Maps UNIQUE constraint, no such table/column, syntax error, 401/429/timeout, file-not-found to plain-language title + hint. Collapsed "Details" shows raw error.

- [x] **Real onboarding empty state** — recent DBs list (last 10, persisted in Zustand store) shown below Open button. Drag-and-drop `.db` onto window opens it. Note: sample DB not bundled (deferred).

- [x] **Window state persistence** — `tauri-plugin-window-state` added to Cargo.toml + registered in `lib.rs`. Window defaults changed to 1100×720 with min 720×480; plugin saves/restores size and position across launches.

- [x] **Wire up or remove dead buttons** — removed Save and copy-icon buttons from tab-bar (no `onClick` implementation was planned).

- [x] **`?` shortcut sheet** — `ShortcutSheet.tsx` modal with all shortcuts grouped by section. `?` key (no modifier) toggles it from anywhere except inputs. Esc/`?` closes it.

- [x] **Accessibility minimum**
  - `SettingsModal.tsx`: Esc closes, focus trap, returns focus to trigger element on close, `role="dialog"` + `aria-modal` + `aria-label`
  - Close buttons have `aria-label`
  - `ShortcutSheet.tsx` has same focus trap/Esc/return-focus pattern
